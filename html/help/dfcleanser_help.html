Pandas Dataframe Cleanser

A utility to prepare your pandas dataframe for data analytics.





    System Environment



System Environment is defined to provide info to verify system data such as libraries and setup notes. It also provides reset of the notebook.

  • Select Chapeters allows you to select which dfcleanser chapters to display.
  • Reset Chapeters resets the chapter displays.
  • Clear Data resets all chapters and drops all data.
  • System displays system notes and info.
  • dfcleanser files supports notebook specific files that contain dfcleanser.
  • About displays dfcleanser contact info.
  • EULA displays the End Users License Agreement.


Select Chapters



The Select Chapters functionality allows you to decide which dfcleanser chapters are loaded into your notebook.

You can dynamically load and unload dfcleanser Chapters. The core dfcleanser chapters can not be swapped.



Select the chapters to load and hit the 'Select Chapters To Load'.



Reset Chapters

'Reset Chapters' resets the display of all the chapters and the associated help. Reset does not impact dataframe content.

Clear Data

'Clear Data' resets the display of all the chapters and clears the working dataframe('default') with any changes resulting in an empty working dataframe.

System



'System' displays a set of notes and setup info for the user to find info about the notebook installation.

'System' displays a list of libraries and their versions to review your environment against dfcleanser.



Green indicates the libs match.

Yellow indicates the installed lib is newer than the tested with lib which should be ok.

Red indicates the installled lib is older than the tested with version which could cause unknown issues



dfcleanser files



The dfcleanser files functionality allows you to manage the notebook specific dfcleanser files(cfg and script).

If you copy, rename or delete a notebook that has dfcleanser run witin it you need to handle the associated dfcleanser files.



The 'Notebook Name' parameter defines the existing notebook name.

The 'New Notebook Name' parameter defines the new notebook name.

The 'Copy Notebook' function creates new cfg files for the 'New Notebook Name' from the 'Notebook Name' cfg files.

The 'Rename Notebook' function renames the existing cfg files for the 'Notebook Name' to the 'New Notebook Name' cfg files.

The 'Delete Notebook' function deletes the existing cfg files for the 'Notebook Name'. The 'New Notebook Name' is ignored.



About

About displays the dfcleanser contact info.



EULA

'EULA' displays the End User License Agreement and should be read and acknowledged. The user must read and agree to continue.

    Data Scripting





The scripting mode functionality allows you to keep a running log of all of the commands run within the Dataframe Cleanser that import, modify or export the current dataframe.

Only commands that modify the content of the dataframe are stored in the scripting log. Any commands that display results like Data Inspection are not stored in the script log.

Scripting allows you to recreate cleansed dataframes from the same input source or to generate cleansed dataframes that are periodically regenerated with newer data.

You can modify the script log to customize it and rerun the same commands on similiar but different data sets.

Calculate Distance



Calculate allows you to interactively calculate a distance between two locations.





The from_address and to_address parameters can be defined as an address or coordinates. For an address use a string value. For coordinates use a list or tuple of Latitude or Longitude.

For the 'distance_algorithm' and 'elipsoid' parameters you can open the dict utility and review appropriate values.



Display Bulk Distance





The 'from_column' and 'to_column' parameters define the address or coordinates columns to calculate distance between.

For an address use a string column or build a list of column names to use in order as the address string.

For coordinates use column names containing a list or tuple of Latitude and Longitude.

You can also define a list of lists for when latitude and longitude are in separate columns.

Latitude and Longitude in same column : '*_column' = column name

Latitude and Longitude in unique columns : '*_column' = [latitude column name, longitude column name]

You must select a column name for the calculated distance to be stored in.

    Dataframe Subset



The Dataframe Subset Utility allows you to select subsets of the current dataframe.



Get Dataframe Subset

The Get Dataframe Subset screen allows the user to enter the parameters needed to extract a subset.



The drop_row_range parm defines the slice of the dataframe that is to be dropped before processing any filters. The default is None so no rows are dropped.



The column_names_list field defines the list of columns to keep or drop for the subset.

The column name values can be selected by clicking on names in the 'Column Names' table or entered by hand (comma separated).



The keep_column_names_flag determines if the names in the column_names_list are to be kept in the new subset or dropped from the new subset.

The default value of True indicates to keep the column names and False indicates to drop the column names.

The csv_file_name parm allows the user to save the resultant subset in a csv file.

Once you select the parms you can add a filter for the subset or get the subset without filters.



Dataframe Subset Filters



The DFSubset Filter function allows the user to define a filter against the dataframe for selecting rows based on column values.

The previouslty defined parms for row slice, column list and file name are displayed for info purposes.



Each filter is defined by the column_name the filter is applied against.

Each filter can have multiple subclauses defined for the filter.

All '==' subclauses are joined in one subclause with == or == or == .....



The following example shows how to define filters.

The first filter defines a range of values 'age >= 11.0' and "age <=4 5.0 ".

the second filter defines the condition of 'ba==not present '.



Dataframe Subset Filter 1 Example



Initially the user must enter the filter_column_name from the 'Column Names ' table or by hand.



Once the column name is selected you can select values from the 'Column Values ' table.

By selecting a value from the 'Column Values' table a value is put in the filter_column_values field.

A default operator of '==' is automatically put in the column_value_operators field and can be changed as desired.





Both fields can be enetered by hand and a pair of value and operator must be entered.



You can override the default '==' operator by clicking on the '>=' operator from the operators list.





You can now select or enter additional values like '45.0' for filter_column_values.



A default operator of '==' is automatically inserted as the operator.

A default filter column operator of 'or' is inserted in the filter_column_operator(s) field.



You can then change the current operator of '==' to ' <=' using the operators.



You now have a filter defined as ' age>= 11.0 or age <=4 5.0 '.



You can then change the defult filter_column_values(s)_logical by clicking on the 'and' logical.



You now have a filter defined as 'age >= 11.0 and age <=4 5.0 '.

You can then add the current filter by hitting the 'Add Filter To Criteria' key.



Once you add the current filter it shows in the solution_criteria field and all other form fields are reset.

A default value of 'AND' is enerterd into the next_filter_logical field indicating to 'AND' the next filter to the current filter.





Dataframe Subset Filter 2 Example



You can change the inter filter logical from 'AND' to 'OR' via the logial keys.



You can also negate the inter filter logical from 'OR' to 'OR NOT' with the 'NOT' link.



You can then create another filter by selecting a new name from the 'Column Names' table.



For the previously defined example select 'ba' as the next column filter column_name.



You then select the 'notpresent' value and the second filter is now 'ba == notpresent'.



When you finish the second filter add it once again to the selection_criteria via the add filter key.



The second filter is now added to the selection criteria.



Once you finish defining filters hit the Get DF Subset button to get the subset.



Get DF Subset



Once you get the subset status and summary info is displayed.



The status show the number of rows and columns in the new dataframe.
