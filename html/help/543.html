The 'lambda fn flag' indicates a simple function (lamda) to apply against each value in the column.

Use colval in your lambda code as the identifier for each column value.

To apply a more complicated (non lamda) function against the column follow the guidelines below.

Set 'lambda fn flag' to False indicating a complex function.

Be sure to include all imports plus code to run the function.

Use the dfcleanser.common.cfg get_dc_dataframe() to get the dataframe for your function.

Use the dfcleanser.common.cfg get_current_col_name() to get the column name inside your function.

To save the function in the Generic Functions Repository for later usage enter a function_name.

If function_name is blank code is run as is and not stored for later usage.

If you select a function from Generic Functions Repository you can update and it is always saved back.



'column_to_apply_fn_to' defines the column to apply the fn to.

'lamda_fn_flag' defines if the function is an inline lamda function.

'function_name' defines the name to call the function that is stored.

'function_to_apply' defines the code that defines the function.
